# PlainBlack Claude Code Instructions

This repository includes the PlainBlack working strategy, voice and visual rules. Treat these files as operating instructions, not optional inspiration.

Read these before generating copy, interfaces, pages, prompts, components or client-facing artefacts:

1. `docs/01_PlainBlack_Voice_and_Content_Guide.md`
2. `docs/02_PlainBlack_Visual_Brand_Guide.md`
3. `docs/03_PlainBlack_Business_Strategy_Overview.md`

## Non-negotiables

- PlainBlack is a lateral small-business invention studio, not a generic agency with cheeky copy.
- Strategy before decoration.
- Make confusing things clearer.
- Speak to capable but stretched small business owners. They are not stupid. They are stuck.
- Explain the mechanism behind claims.
- Use concrete small-business details.
- Avoid generic AI phrasing, SaaS visuals, vague transformation promises and dependency-led offers.
- Aim humour and swearing at bad systems, not clients.
- Use the calmer register for sensitive subjects.
- Use supplied logo assets manually. Do not regenerate or redraw the wordmark.
- Default visual style: deep black, warm practical light, restrained mint signal, useful artefacts and cinematic grit.
- For visual work, follow the exact typography, spacing, safe-margin, logo-sizing and format presets in `docs/02_PlainBlack_Visual_Brand_Guide.md` before inventing alternatives.
- Do not ship a generic quiz, audit, dashboard or AI wrapper without a memorable lateral move and a useful keepable artefact.

## Before finalising any output

Ask:

1. Is the idea commercially useful?
2. Is the owner respected?
3. Is the mechanism visible?
4. Is there a real specific detail?
5. Does this reduce confusion?
6. Could another agency publish it after swapping the logo?
7. Does it sound or look like AI template sludge?

If another agency could use it unchanged, it needs more PlainBlack thinking.

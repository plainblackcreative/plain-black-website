The 30-Day Build Challenge

Jayden & Ian · May 2026

Thirty products in thirty days. Jay builds, Ian talks. We ship one tool every day from 1 May to 30 May 2026 and post about it on Facebook (the public tracker pulls reach straight from Facebook).

The setup

  • Jay ships one product, tool, or feature each day
  • Ian shows up on camera and tells the story
  • Some are client builds, some are PlainBlack internal tools
  • Every build gets a Facebook post, a video, and a blog post

What this dashboard tracks

  • The build itself: what it is, who it is for, category, price model, where it lives
  • The post: where it ran, likes, leads (anyone who reached out off the back of it)
  • Reach is pulled from Facebook automatically every two hours (see the live number on /challenge), so the tracker form doesn't ask for it
  • A daily checklist so nothing falls off the truck
  • A weekly report on days 7, 14, 21, and 28 with auto-summed totals, fun and enthusiasm ratings, and reflections

The daily flow

  1. Build the tool and deploy it
  2. Write the post copy and visual
  3. Publish the blog post
  4. Post on Facebook (and anywhere else relevant)
  5. Ian records and posts the video

How to use it

  • Click any day in the sidebar to load it
  • Today is auto-selected on open
  • Status dots: mint (Done), amber (In progress), grey (Empty), crossed (Skipped)
  • Field changes save to your browser instantly. Click Save in the top bar to push to GitHub so the other operator can see them
  • The CLIENT pill appears in the sidebar when the Client field is filled with anything other than PlainBlack

The rule

Once it works → stop. Ship it. Move on.

Saving and sharing

Edits save to your browser instantly. To share progress with the other operator, click Save in the top bar — that commits the data to the private plainblack-client repo. On open, the dashboard pulls the latest from GitHub and merges it in.

If both operators edit the same file at once, the second Save will be blocked with a "reload to see the latest" prompt. Reload, your work is preserved in your browser, then re-apply on top of the fresh data.

Notes

Admin only. Requires a GitHub PAT with plainblack-client contents (read/write) to sync. Without a PAT it still works — just stays local to your browser.

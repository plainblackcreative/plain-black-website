Background Rule

Non-negotiable: All customer deliverable playbooks always use a white background. Never dark. The dark header is the only dark element in a playbook.

Layout

  • Sticky dark header with business name and progress bar
  • Left sidebar — 260px sticky — section nav, progress stats, last updated date
  • Main content — white card sections, 16px border radius, soft border
  • Max width: 1200px. Side padding: 32px.
03
Four-File Structure
Same for every niche — what each file does and must contain
▼

File 1 — Landing Page

[niche-slug]-landing.html — generic, not personalised per customer. Sells the product. Nav CTA links to the demo. Form CTA links to Stripe.

Required sections in order:

  1. Sticky nav — logo + CTA button linking to demo
  2. Hero — two-column: headline/desc/CTA left, product preview card right
  3. Proof bar — 4 stats
  4. Problem section — 6 pain cards in a grid
  5. Modules section — all 10 sections as cards with tool pills
  6. AI feature section — explanation left, dark terminal demo right
  7. Reviews — 3 fictional but plausible testimonials
  8. Included + Price — checklist left, sticky price card right
  9. Form section — 3-step process left, Web3Forms form right
  10. Footer

File 2 — Demo / Gated Sample

[niche-slug]-demo.html — lets prospects experience the product. Converts via the paygate modal.

  • Sections 1-2: Fully open, interactive, AI update buttons work
  • Section 3: Partially visible, fades out, pointer-events disabled
  • Sections 4-10: Locked cards with blurred overlay, click fires paygate modal
  • Scroll trigger: Invisible div between Section 3 and teaser card fires paygate on scroll
  • Paygate modal: Lists all 8 locked sections, price card, CTA links to landing page
  • Header badge: "Sample Playbook — 2 of 10 sections unlocked"
  • Progress: Tracks only the 2 free sections — shows "0 / 2"

File 3 — Playbook Template

[niche-slug]-TEMPLATE.html — the master file Claude copies and fills per customer.

Zero placeholder rule: A delivered customer file must contain zero [[PLACEHOLDER]] strings. Claude does a final check before outputting every file.

Section anatomy — every section must contain all of these:

Section card ├── Header: icon · number · title · tool pills · check btn · expand btn └── Body (hidden by default, toggled on click) ├── callout (green) — goal or key principle ├── content-grid — 2 columns of content-cards with → arrow lists ├── callout-warn (gold) — sanity check or common mistake ├── diy-accordion — numbered steps, collapsible ├── checklist — interactive tick items, localStorage persistence └── AI update button + dark response box
No upsell block. No ongoing offers. The product is self-contained and complete.

File 4 — Project Instructions

PROJECT_INSTRUCTIONS.md — paste into the Claude Project's Instructions field. Every chat in that project inherits this automatically.

  • What the project does (one paragraph)
  • The four files table
  • Design system rules
  • 10 sections with titles and tools
  • Section anatomy
  • Tool pill colour coding
  • AI system prompt pattern
  • Per-customer customisation checklist
  • Writing tone rules
  • Trigger phrase and worked example
  • Final check: zero placeholders in delivered file

05
Writing Tone
Applies to every niche, every file, every section
▼
  • Direct, no-fluff, second-person — "you should", not "one should"
  • Reads like an experienced person sitting next to them, not a marketing guide
  • Short sentences. No waffle. No preamble.
  • No em dashes — use commas, semicolons, or periods instead
  • No generic AI voice — every sentence should feel specific to this customer's business
  • Sanity checkpoints at key stages: "If your screen doesn't look like this, stop"
  • Specific beats vague — name the tool, give the time estimate, state the cost
  • Common mistakes sections build trust fast
  • The playbook is a system, not a guide — checklists, sequences, decision points
No em dashes anywhere. This rule applies to all content, all niches, all files. Use commas, semicolons, or periods.

AI Update Button System
Identical pattern across all niches — only the prompts change
▼

Every section in every playbook has a "Get Current Advice" button. The API call pattern is identical across all niches. Only the system prompt and per-section prompt change.

[Other relevant context from placeholders.]
Give specific, actionable, current advice. No fluff. No generic tips.
Maximum 250 words. Direct second-person. No em dashes.`;

/* API call — identical across all niches */
const res = await fetch('https://api.anthropic.com/v1/messages', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    model: 'claude-sonnet-4-20250514',
    max_tokens: 1000,
    system: SYSTEM_PROMPT,
    tools: [{ type: 'web_search_20250305', name: 'web_search' }],
    messages: [{ role: 'user', content: sectionSpecificPrompt }]

Web search is always enabled. This keeps every AI response current regardless of when the customer opens their playbook.

08
Starting a New Niche Project
Four steps from brief to live project
▼

Step 1 — New Claude chat (not a project yet)

  1. Paste this full bootstrap kit document
  2. Immediately paste your completed Niche Brief (Section 1 filled in)
  3. Say: "Build the four deliverables for this niche. Start with the landing page."
  4. Claude builds in order: landing page, demo, template, project instructions

Step 2 — New Claude Project setup

  • Project name: e.g. "PlainBlack — Google Reviews Playbook"
  • What are you trying to achieve: "Build personalised, interactive HTML [niche] playbooks for [customer type]. Each chat takes a business's details and outputs a complete single-file HTML playbook ready to deliver."
  • Project Instructions field: Paste the full contents of the generated PROJECT_INSTRUCTIONS.md
  • Project Knowledge: Upload all four generated files

Step 3 — Test the trigger

In a new chat within the project, type the trigger phrase with a real or fictional business. Confirm Claude builds a complete personalised file with zero [[PLACEHOLDER]] strings remaining before treating the project as live.

Step 4 — Archive the bootstrap chat

Download all four files. The bootstrap chat can be closed. Everything lives in the project.

10
Placeholder Convention
Same across all niches — zero placeholders in any delivered file
▼
  • All placeholders use [[DOUBLE_BRACKETS_CAPS_UNDERSCORE]] format
  • They appear in: HTML content, the CSS comment block at top, JS SYSTEM_PROMPT, JS PLAYBOOK_ID, and per-section AI prompts
  • The CSS comment block at the top of every TEMPLATE file lists every placeholder with an example value
  • Common to all niches: [[BUSINESS_NAME]] [[BUSINESS_SLUG]] [[CITY]] [[REGION]] [[COUNTRY]] [[MONTH_YEAR]]
  • Niche-specific placeholders are defined in the Niche Brief — Section 1
Final check rule: Before outputting any customer file, Claude searches the full HTML for any remaining [[ string. If found, it completes them before delivering. A file with placeholders is never a finished deliverable.
